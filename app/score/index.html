Mahjong Scoring Helper — Hong Kong style

You (the chatbot) have been asked to help someone score a Hong Kong mahjong hand. Your job is to transcribe the hand, gather the win context, then produce a clickable URL that renders the scoring result. Do not fetch the URL yourself — output it as a link and the user clicks it to see the score in their browser.

Workflow

  1. Ask the user for a photo (or description) of their winning hand.
  2. Identify each tile using the encoding below.
  3. Group tiles into melds. A standard winning hand is 5 melds: 4 sets (chow / pong / kong) plus 1 pair. Flowers/seasons go in a separate flower meld counting them. Special shapes: 13 orphans (one orphans meld), or 7 pairs (seven pair melds).
  4. Ask the user for the win context you can't see in the photo: who won, who's dealer, self-pick vs discard (and who discarded), how many consecutive rounds the dealer has won, and any special conditions.
  5. Assemble the JSON payload (shape below).
  6. URL-safe base64 encode the JSON (base64url: - and _ instead of + and /; strip = padding).
  7. Tell the user: "Click this link to see the score: <origin>/?d=<BASE64>" — note this points to the app root (/), not /score. The URL opens the Mahjong Scorer app with the hand pre-loaded, ready to show the score (and editable if the user spots a mistake).

Meld schema

type MeldType = 'chow' | 'pong' | 'kong' | 'pair' | 'flower' | 'orphans';

interface Meld {
  type: MeldType;
  tiles: Tile[];       // chow=3 consecutive same-suit, pong=3 identical,
                       // kong=4 identical, pair=2 identical,
                       // flower=N copies of 'F' (N = flower count),
                       // orphans=14 tiles (13 terminals/honors + the paired one)
  concealed: boolean;  // true = hidden in hand, false = exposed on table.
                       // flowers are always exposed; set false.
  winTile?: Tile;      // the tile that completed the hand, if in this meld
}

Win schema

interface Win {
  players: [string, string, string, string];  // four player names/labels
  winner: string;                              // one of players
  dealer: string;                              // one of players
  method: 'self-pick' | 'discard' | 'stolen-kong';
  from?: string;                               // required unless self-pick
  dealerRounds: number;                        // consecutive rounds dealer has won (>=1)
  special: ('fromButt' | 'lastTile' | 'firstTurn' | 'prodigy')[];
}

Payload shape

{
  "hand": { "melds": [ /* Meld[] */ ] },
  "win":  { /* Win */ }
}

Worked example

Chow 1-2-3 bamboo, chow 4-5-6 bamboo, chow 7-8-9 dots (won on 8d), exposed red dragon pong, pair of 5 bamboo. Player A won; B is dealer; D discarded the winning tile.

{
  "hand": {
    "melds": [
      { "type": "chow", "tiles": ["1b","2b","3b"], "concealed": true },
      { "type": "chow", "tiles": ["4b","5b","6b"], "concealed": true },
      { "type": "chow", "tiles": ["7d","8d","9d"], "concealed": true, "winTile": "8d" },
      { "type": "pong", "tiles": ["Rd","Rd","Rd"], "concealed": false },
      { "type": "pair", "tiles": ["5b","5b"], "concealed": true }
    ]
  },
  "win": {
    "players": ["A","B","C","D"],
    "winner": "A", "dealer": "B",
    "method": "discard", "from": "D",
    "dealerRounds": 1, "special": []
  }
}

Base64url-encoded into a URL (click to open the Mahjong Scorer with this hand pre-loaded):

/?d=eyJoYW5kIjp7...

Notes for AI callers

  • Do not fetch this URL yourself. Output it as a link for the user to click.
  • Tile shorthand: lowercase suit letter, uppercase honor letter. Don't invent variants.
  • If a tile is ambiguous, ask the user to describe it or send a clearer photo.
  • If win context is missing (winner, dealer, method), ask the user — don't guess.
